Welcome to the AppArena.com App-Template

This is only a Testpage and is called over the demo.js in /js/ directory. In this file is ja DemoView initialization and a render function call, that calls this templates what you see now. The /js/ directory is the App main folder. Here you find all app stuff. Views, models, collections, modals, vendor stuff, thr router and so on. Modules are installed over bower in the root (/source/) ./modules/ directory.

The app template includes a Vagrant configuration. That means, you can start this (it's an VirtualBox image) with a complete server configuration and start develop, without any locale machine configuration. The Vegrant configuration includes an Ubuntu minimal server with PHP 5.3, MySql Server, Apache Webserver, xDebug, Xhprof and some envirement settings.

Pre-Configuration and install needed sources

Install Bower, equirejs and r.js on windows:

  1. Downlaod nodejs and install it from: http://nodejs.org/
  2. Open a GIT console and enter
    • npm install requirejs -g
    • npm install bower -g

Get vendor sources:

Open a GIT console, change into the app sources directory and enter: 

bower install

This will install install all depended sources for the main aa-app-template. PLease pay attention, that you change the version numbers in the dependencies source file, when you finished development. Take a look into the file bower.json in the root directory and configure that for your app.

Compile sources:

Open a GIT console, change into the app sources/build directory and enter: 

sh ./build.sh

After that, the r.js compiler starts the compiling process and shrings your app. The result is stored in /dist/. If you use new modules or build some new ones by your own, please configure them in the building file, that is called /build/app.build.js

Install new app packages:

Open a GIT console, change into the app sources directory and enter: 

bower install <PACKAGENAME> or <URL TO ZIP PACKAGE>

To save the package to the app install/update process enter: 

bower install <PACKAGENAME> or <URL TO ZIP PACKAGE> --save
